## 1. Вербальное описание

**Тема:** Анализ программных проектов с использованием метрик качества

**Проблема:** Современная индустрия ПО переживает глобальный кризис качества, который угрожает экономической стабильности бизнеса. Утечки памяти достигают катастрофических масштабов: калькулятор Apple теряет 32 ГБ RAM, VS Code — 96 ГБ через SSH, Spotify потребляет 79 ГБ, Discord — 32 ГБ за минуту демонстрации экрана. Системные сбои стали нормой: обновления Windows 11 ломают меню "Пуск", Spotlight записывает 26 ТБ на SSD за ночь, Android 15 выпущен с 75+ критическими уязвимостями. Инцидент CrowdStrike 2024 года из-за отсутствия проверки границ массива привел к падению 8,5 млн ПК и убыткам в $10 млрд. ИИ усугубляет ситуацию: генерируемый код содержит на 322% больше уязвимостей, 45% всего сгенерированного ИИ кода включает эксплуатабельные дыры. Разработчики используют разрозненные инструменты без единой оценки качества, что приводит к накоплению технического долга и рискам production-сбоев.

**Назначение системы:** Веб-приложение для комплексного анализа программных проектов предназначено для автоматической оценки качества кода, безопасности и бизнес-показателей с предоставлением конкретных рекомендаций по улучшению. Система обеспечивает разработчикам единое окно с тремя видами анализа проекта, предотвращая экспоненциальную деградацию качества ПО. Платформа рассчитана на разные команды разработчиков, в которых нет, или только появляется культура чистоты и безопасности кода.

**Суть деятельности:** Пользователь загружает проект в систему, которая автоматически анализирует код с помощью статических анализаторов (Ruff, Radon, Bandit, Semgrep), проверяет покрытие тестами и ресурсные утечки, выявляет технический долг с помощью ИИ, формирует отчеты по стандартам OWASP Top 10, CWE Top 25 и OWASP ASVS. Система агрегирует бизнес-метрики (DAU/WAU/MAU, CSI, LeadTime) и рассчитывает коэффициент полезных коммитов, предупреждая о проблемах качества до production-инцидентов. Приложение возврпредоставляет пользователю развернутый отчет о его проекте.

**Внешние обстоятельства функционирования:** Платформа должна уметь работать с проектами разного размера, структуры. Система должна быть масштабируема, отказоустойчива.


**Основы функционирования:** Архитектура системы включает три модуля с автоматическим мониторингом качества на основе метрик:

**Модуль качества кода:**

Набор программных показателей, которые позволяют разработчикам лучше понять код:

- **Коэффициент полезных коммитов**:

$$
    Q_{fx} = 100\\% × (C_{fx} / C_{total})
$$

    Где:  
    $Q_{fx}$ — коэффициент полезных коммитов (полезных/всего)  
    $C_{fx}$ — число коммитов, связанных с разработкой продукта  
    $C_{total}$ — общее число коммитов, сделанных за перио  

- **Code coverage** — процент кода, покрытого тестами на предмет ошибок и багов
- **Crash-Free Users (%)**, или отказоустойчивость — процент пользователей, которые не столкнулись со сбоями приложения
- **Mean Time Between Failures (MTBF)** — показывает среднее время между инцидентами отказов программы
- **Mean Time to Repair (MTTR)** — среднее время восстановления системы после сбоя
- **Техдолг** - определение с помощью AI-агента, обученного на открытых кодовых базах

- [Ruff](https://astral.sh/ruff)
- [Radon](https://radon.readthedocs.io/en/latest/intro.html) (оценивает сложность кода используя метрики Холстеда, Индекс поддерживаемости, сырые метрики)


**Модуль безопасности:**
Формирование трех отчетов, аналогично функционалу SonarQube:

- [OWASP TOP 10 report](https://owasp.org/Top10/2025/)
- [Vulnerabilities and Security Hotspots conforming to the CWE Top 25 report](https://cwe.mitre.org/top25/index.html)
- [OWASP ASVS - Vulnerabilities and Security Hotspots conforming to the OWASP Application Security Verification Standard](https://owasp.org/www-project-application-security-verification-standard/)

**Статический анализ кода:**
- [**Bandit**](https://bandit.readthedocs.io/)
- [**Semgrep**](https://semgrep.dev/)

Возможно использование [CVSS](https://www.securitylab.ru/analytics/355336.php)

**Модуль бизнес-метрик:**
Для проектов, которые не могут быть оценены с точки зрения финансов:

- **DAU/WAU/MAU** - Daily/Weekly/Monthly active users
- **CSI** - индекс удовлетворённости клиентов
- **LeadTime** - общее время, прошедшее с момента, когда задача была запрошена, до её полного завершения и передачи заказчику

**Можно определить дополнительные метрики при необходимости.**

**Хранилище данных** обеспечивает хранение истории анализов, трендов метрик и рекомендаций для отслеживания прогресса улучшения качества проектов.
